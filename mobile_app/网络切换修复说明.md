# 网络切换地址更新修复说明

## 问题描述
导入助记词后，切换到TRON网络时地址不变。

## 根本原因
当用户切换网络时，前端只是更新了`_currentNetwork`变量，但没有重新调用后端API来根据同一助记词生成新网络的地址。

不同区块链使用不同的地址派生路径：
- **TRON**: `m/44'/195'/0'/0/0` → T开头的地址
- **BSC/ETH**: `m/44'/60'/0'/0/0` → 0x开头的地址（BSC和ETH共用同一地址）

虽然私钥相同，但地址格式和生成算法不同。

## 修复方案

### 核心思路
**导入助记词时一次性生成2个网络的地址并保存到本地，切换网络时直接从本地读取，无需调用后端API。**

### 实现细节

#### 1. 导入时自动生成双网络地址
在 `_finalizeWallet` 函数中，保存当前网络钱包后，自动调用 `_generateOtherNetworkWallet`：

```dart
// 如果有助记词，自动生成另一个网络的地址
if (mnemonic.isNotEmpty) {
  await _generateOtherNetworkWallet(mnemonic, privateKey);
}
```

`_generateOtherNetworkWallet` 逻辑：
- 如果当前是TRON → 生成BSC地址（BSC/ETH共用）
- 如果当前是BSC/ETH → 生成TRON地址
- 检查本地是否已存在，避免重复生成

#### 2. 切换网络时从本地加载
在 `didUpdateWidget` 中监听network变化：

```dart
@override
void didUpdateWidget(WalletManagementPage oldWidget) {
  super.didUpdateWidget(oldWidget);
  // 当网络切换时，如果有助记词则从本地加载对应网络的地址
  if (oldWidget.network != widget.network && (_mnemonic?.isNotEmpty ?? false)) {
    _loadWalletForNewNetwork();
  }
}
```

`_loadWalletForNewNetwork` 逻辑：
- 从本地存储查找对应网络的钱包
- BSC和ETH共用同一个钱包数据
- 找到则直接加载，无需网络请求
- 如果session_id失效，自动刷新
- 如果未找到，才调用后端API创建

## 技术细节

### 地址派生路径
```
助记词 → Seed → BIP32 Master Key
                    ↓
            根据不同链选择coin_type:
            - TRON: coin_type = 195
            - ETH/BSC: coin_type = 60
                    ↓
            m/44'/coin_type'/0'/0/0
                    ↓
            私钥 → 公钥 → 地址
```

### 后端实现
`wallet_api_service.py` 中的 `import_mnemonic` 函数：
```python
wallet = MultiChainWallet(chain=chain, mnemonic=mnemonic)
```
MultiChainWallet 构造函数会根据chain参数自动选择正确的派生路径。

### 前端存储
助记词保存在本地SharedPreferences中：
```dart
await WalletStorage.saveWallet({
  'chain': widget.network,
  'address': address,
  'private_key': privateKey,
  'mnemonic': mnemonic,  // 关键：保存助记词
  'session_id': sessionId,
});
```

## 注意事项
1. 只有导入过助记词的钱包才会在切换网络时自动更新地址
2. 通过私钥导入的钱包不会自动切换（因为没有助记词无法派生其他链的地址）
3. 每次切换网络都会创建新的session_id
4. 云端备份会为每个网络分别保存（因为地址不同）
